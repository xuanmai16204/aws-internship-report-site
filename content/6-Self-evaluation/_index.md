# Self Assessment

## Internship Experience

During my internship from April 17, 2026, to July 10, 2026, at Amazon Web Services Viet Nam Company Limited, I had a valuable opportunity to participate in developing the CloudDoc project—an online document management and sharing system for students. As a team member, my primary responsibility was designing and implementing the user interface (Frontend) using React. Additionally, I studied and assisted in backend integration, including analyzing document metadata storage structures, and researched the project's AWS cloud architecture, which features S3, CloudFront, EC2, and RDS. Working within a professional enterprise environment has significantly built my professional mindset, teaching me how to systematically analyze and solve complex technical challenges.

## Skills Improved

Through this internship, I have cultivated and enhanced several practical capabilities:
- **UI/UX Design and Optimization**: Built user-centric, responsive, and friendly interfaces for essential application modules, such as Home, Search, Upload, Live Preview, and the Administration Dashboard.
- **Reading Technical Documentation**: Improved my ability to read and exploit official AWS developer guides, backend API documentation, system architecture patterns, and data flow diagrams.
- **Working with APIs**: Acquired hands-on experience in connecting, requesting, and integrating RESTful APIs developed in Node.js/Express.
- **Understanding Frontend/Backend Integration**: Understood the distinct responsibilities of server-side logic and the user interface, notably managing asynchronous tasks and optimizing content delivery performance.
- **Self-learning through Workshops and AWS Documentation**: Strengthened my capability to search for solutions independently through authoritative documentation, actively participate in internal workshops, and discuss technical details with colleagues.

## Strengths

### Proactiveness
I demonstrated strong initiative by actively analyzing requirements and taking ownership of front-end UI tasks from the initial phase of the project. Whenever I encountered new features or unfamiliar mechanisms, such as configuring direct S3 uploads via presigned URLs, I conducted detailed research and built mock prototypes before presenting solutions to the team.

### Learning and Adaptability
Operating in a real cloud environment with AWS services initially posed a learning curve. However, by asking questions during technical discussion sessions and actively consulting documentation, I quickly adapted to the system architecture and adjusted the frontend client interface to coordinate properly.

### Team Collaboration
I constantly communicated with backend team members to align on metadata schemas, HTTP API payloads, and project milestone objectives. This effective coordination ensured that user interfaces, document upload workflows, and moderation dashboards were integrated smoothly and on schedule.

## Areas for Improvement

### System Thinking
I need to expand my perspective beyond the frontend client code to better understand the backend, database, and infrastructure integration, allowing me to propose more comprehensive, full-stack solutions.

### Technical Presentation Skills
I hope to improve my communication skills to express technical architectures and frontend implementation concepts more concisely and persuasively during sprint reviews and team alignments.

### Hands-on AWS Experience
Although I have acquired a strong conceptual understanding of AWS cloud architectures, my hands-on configuration skills (such as managing VPC routing or structuring security policies) require further practice on the AWS Console.

## Reflection

### Challenges
The primary challenge arose when shifting from mock data structures to active backend endpoints. Syncing document metadata and handling client-side uploads using S3 presigned URLs required precise CORS and parameter settings, which led to initial connection issues. Furthermore, gaining familiarity with the overall AWS deployment architecture required a significant investment of time and research.

### Solutions
To overcome these issues, I read the official AWS documentation and reviewed hands-on guidelines from previous workshops. I also coordinated closely with backend developers to log errors and break down complex integration tasks into minor, testable units that could be implemented step-by-step.

### Future Development
Moving forward, I intend to develop my skills as a Fullstack Engineer, balancing frontend design with robust server-side development. I also aim to study cloud-native deployment patterns on AWS, modern application monitoring tools, and optimization principles to build reliable, scalable web applications.

## Conclusion

My internship at Amazon Web Services Viet Nam Company Limited from April 17, 2026, to July 10, 2026, has been an invaluable milestone in my academic and professional path. Building the CloudDoc system helped me gain confidence in my frontend development capabilities while giving me valuable exposure to cloud technologies. This experience serves as a solid foundation for my future career in computer science.
