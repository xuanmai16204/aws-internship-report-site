#### Objectives

Before starting the document upload workflow and system testing, the deployment environment should be fully prepared. Verifying the required components in advance helps reduce unexpected issues during the implementation process.

#### Components to Verify

The following components should be checked before continuing:

- The web application is accessible.
- The backend API is running normally.
- The Amazon S3 bucket has been created and configured.
- Amazon RDS PostgreSQL is available for connections.
- Sample data is prepared for testing.

#### Purpose of the Preparation

Checking these components beforehand ensures that the environment is ready for the remaining workshop tasks. This also makes it easier to verify the outcome of each implementation step.

#### Implementation Steps

1. Verify the deployment environment.
2. Prepare sample data and system configuration.
