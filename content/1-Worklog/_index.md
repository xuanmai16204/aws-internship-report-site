This section summarizes my twelve-week internship journey in the **First Cloud AI Journey** program. Each weekly worklog follows the same structure to document learning progress, practical activities, and project development throughout the internship.

The worklogs are organized chronologically, beginning with AWS fundamentals, continuing through workshops and technical research, and ending with the implementation and completion of the CloudDoc project.

The internship timeline is summarized below:

**Week 1:** Set up the working environment and explore basic AWS services.

**Week 2:** Practice with Amazon EC2, Amazon S3, and AWS Identity and Access Management (IAM).

**Week 3:** Learn Amazon Virtual Private Cloud (Amazon VPC) and networking concepts.

**Week 4:** Participate in company activities and start the CloudDoc project.

**Week 5:** Study frontend architecture and technical solutions.

**Week 6:** Join workshops, seminars, and technical events.

**Week 7:** Continue exploring AWS architecture and prepare project resources.

**Week 8:** Develop the user interface and implement core CloudDoc features.

**Week 9:** Improve the interface and complete sample data.

**Week 10:** Implement authorization and document moderation features.

**Week 11:** Finalize preview, download, and project demonstration.

**Week 12:** Perform system testing, integrate AWS backend components, and complete the internship report.
