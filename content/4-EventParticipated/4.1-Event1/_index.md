# SUMMARY REPORT
# AWS Community Day – First Cloud AI Journey

### 1. General Information
- **Event Name:** AWS Community Day – First Cloud AI Journey
- **Date & Time:** May 23, 2026
- **Location:** Amazon Web Services Viet Nam Company Limited Office
- **Role:** Participant (Attendee)

### 2. Event Objectives
The objective of AWS Community Day was to equip student interns with solid theoretical and practical foundations in Cloud Computing and Generative AI on AWS. The event also served as a crucial orientation milestone to guide project ideas, layout system architectures, and foster collaboration for the CloudDoc project.

### 3. Key Activities
During the event, I actively participated in several technical presentations and collaborative break-out sessions:
- **Session 1: Build Second Brain – Context Is Everything**: Learned the importance of supplying rich context when working with AI to improve output quality, and explored personal learning repository management.
- **Session 2: Friendly AI Assistant with Amazon Q**: Observed live demonstrations of using Amazon Q as an intelligent IDE assistant to write code, debug errors, and explain framework files.
- **Session 3: From Edge to Origin – CloudFront as Your Foundation**: Explored how static content is distributed through CDN locations using Amazon CloudFront and optimized with proper caching.
- **Session 4: Deep Dive Talk – How LLM Actually Works**: Learned the mechanics of Large Language Models, including token output predictions and formatting settings like Temperature.
- **Session 5: CloudDoc Project Orientation**: Participated in an alignment discussion with AWS mentors to define Git workflows, component abstractions, and integration strategies.

### 4. Knowledge Gained
Through the AWS Community Day session, I gained several key technical insights:
- **AWS Service Operations**: Acquired a direct understanding of how EC2, S3, and CloudFront interact to deliver static frontend assets fast and secure.
- **Context and Prompt Engineering**: Realized how to draft high-quality prompts by supplying appropriate context, helping me code and troubleshoot efficiently during standard development tasks.
- **Collaborative Project Alignment**: Discovered best practices in splitting development workloads across team members to build the CloudDoc solution.

### 5. Personal Reflection
This event was highly engaging and practical. It provided me with a valuable opportunity to observe how enterprise solutions are designed on AWS while building my confidence to kick-start the development phase of CloudDoc. Interacting directly with professional cloud developers motivated me to deepen my engineering capabilities.
